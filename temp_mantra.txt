ABSTRACT



Mantra is a comprehensive educational and productivity platform designed to enhance student potential through gamification and integrated study tools. Developed using the Laravel web framework, Mantra provides users with an engaging environment to manage their academic tasks, notes, and schedules interactively.



The platform offers a suite of core features, including a dynamic Chat Interface for real-time collaboration and note-sharing, a gamified User Profile tracking system that rewards consistent study habits with Experience Points (XP), study streaks, and achievements, and a Smart Notes feature capable of automatically extracting and organizing text from uploaded PDF and DOCX documents. Additionally, an interactive Calendar mechanism allows users to easily manage new tasks, while the entire system tracks and visually displays total study time and academic milestones. By combining the social elements of real-time communication via WebSockets with robust productivity features, Mantra creates a streamlined, rewarding, and highly effective digital workspace that encourages continuous learning and collaboration among students.

INTRODUCTION



Mantra is an innovative educational productivity platform designed to foster a more engaging and effective learning experience. By blending traditional study utilities such as calendars, note taking, document management, and communication tools with powerful gamification elements, Mantra aims to transform the way users interact with their daily academic tasks.

In today’s fast-paced digital learning environment, staying organized and maintaining motivation are significant challenges. Mantra addresses these challenges by structurally rewarding consistent study habits. Users earn Experience Points (XP), maintain consecutive activity streaks, and unlock profile achievements as they utilize the platform to study and collaborate.

Developed as a highly responsive web application using the Laravel 12.x PHP framework (operating on PHP 8.2), Mantra ensures a secure and scalable backend architecture. Its diverse frontend is constructed using modern HTML5, CSS3, and vanilla JavaScript for maximum user interactivity. From an integrated interactive calendar to manage upcoming assignments, to real time chat powered by WebSockets for instant student collaboration, Mantra provides a comprehensive suite of utilities. Furthermore, Mantra implements an advanced “Smart Notes” system, allowing users to upload documents (such as PDFs and DOCX files) and instantly automatically extract the content into editable, shareable notes. By providing an all-in-one companion that motivates, organizes, and socially connects learners, Mantra aims to redefine modern academic task management and user engagement.





SUPPORTING LITERATURE



Overview

Building a modern web application like Mantra requires standing on the shoulders of robust, open-source literature and documentation. In the development of this educational productivity platform, several key framing documents, architecture guides, and API specifications were deeply analyzed.



Laravel Documentation

The primary literature driving the backend systems of Mantra is the official Laravel 12.x Documentation. This comprehensive guide dictates the structural MVC routing, Eloquent ORM relationships, and Blade templating methodologies used to render the gamified profile and calendar.



WebSocket Specifications

To achieve real-time, bidirectional communication for the Chat module without continuous HTTP polling, the RFC 6455 WebSocket protocol specification was studied. This literature outlines the handshake process, payload masking, and persistent connection strategies. Implementations like Pusher and Laravel WebSockets strictly adhere to these guidelines to broadcast messages instantly to students collaborating online.



Document Parsing Libraries

The Smart Notes module necessitates accurately extracting raw text from PDFs and DOCX files. Literature regarding PHP document processing libraries (such as Spatie’s PDF-to-Text and PHPWord documentation) provided the foundational methodology for reading file streams, decoding proprietary formats, and pushing translated strings into the database relationally.





SYSTEM ANALYSIS



Module Description

The “Mantra” productivity and study platform is divided into multiple functional modules, each playing a key role in ensuring a smooth user experience, promoting learning, and incorporating gamification. The main modules include:

Gamification & Profile Module: This module is responsible for keeping the user motivated and tracking their academic progress over time. It visually displays critical study metrics, including user achievements, rewards, study streaks, total study time, and Experience Points (XP). By translating time spent studying into quantifiable rewards, it encourages users to stick to their academic tasks.

Calendar & Schedule Module: This module serves as the organizational hub for the student. It visually lays out assignments, ongoing tasks, and goals on an interactive calendar. Recent updates feature refined user experiences, such as displaying a dynamic loading indicator when new tasks are added to prevent UI freezing and confusion.

Real-Time Chat & Collaboration Module: This module is vital for student socialization. It leverages WebSockets to enable instant, real-time message delivery between peers without requiring page reloads. Moreover, this module includes unique features such as a “Share Note” modal, which lets users select a previously written Smart Note and share its content to their friends directly within the chat interface. It also accommodates a music widget to promote focused study sessions.

Smart Notes & File Extraction Module: This module addresses document organization and content generation. Users can securely upload documents (such as PDF and DOCX files), which are scanned by the system. The module extracts the internal text and presents it via a “Save to Notes” dialog. Users can organize these extracted texts into specific folders, seamlessly transforming static files into editable, shareable Smart Notes.

Progress & Analytics Module: Working closely with the Gamification Module, this module manages the accurate calculation and visual display of a user’s total active productivity time. It handles data edge cases naturally; for instance, ensuring that values of exactly zero are properly reflected on the dashboard rather than appearing missing or leading to null errors.





Business Rules

Business rules define how processes and activities should be conducted to ensure consistency across the application.



Note Extraction & Management

All user-uploaded files must adhere to supported extensions (e.g., PDF, DOCX, TXT) to execute successfully through the Smart Notes text extraction engine.

Extracted files must prompt the user with a mandatory folder selection dialog before being formally instantiated as an editable Note.

Collaboration Mechanics

Chat messages must be pushed via WebSockets connecting participants instantly. Users must be authenticated to actively send and receive messages.

To share notes, the user must own the note, ensuring data privacy and preventing unauthorized access to others’ notes.

Gamification & Tracking

Experience Points (XP) are strictly calculated based on the execution of academic tasks, creation of notes, and accumulation of total study time.

Study streaks increment purely based on active daily interaction with the core studying components, promoting a strong learning habit.

Analysis of Architecture

Laravel Framework (PHP 8.2)



Mantra is primarily built on the Laravel 12.x PHP framework, utilizing an MVC (Model-View- Controller) architecture to guarantee clean, scalable, and secure application code.



Backend Architecture (MVC):

The Model layer manages the data relationships inside the PostgreSQL database, ensuring data integrity for things like user notes, milestones, and shared files.

The Controller layer processes business logic, such as extracting text from uploaded PDFs and calculating XP based on actions.

The View layer employs Laravel’s Blade templating engine for assembling dynamic and responsive web pages.

WebSocket Real-Time Engine:

To achieve instantaneous message delivery inside the Chat Module, WebSockets are utilized to bypass traditional HTTP polling constraints.

Data Security & Storage:

File upload streams and extracted note files are managed securely by the Laravel storage facilities, preventing malicious payloads or exposure to unauthorized users.



Key Algorithms

The following algorithms drive Mantra’s core computational logic:

XP Calculation Algorithm



Experience Points are awarded based on discrete academic actions. Each qualifying event (e.g., creating a note, completing a calendar task) triggers an increment to the user’s cumulative XP field in the database. The algorithm queries the user’s current XP, applies the reward delta, and persists the updated value atomically via Eloquent ORM to prevent race conditions.

Streak Tracking Algorithm



The streak counter evaluates whether the student has interacted with any core study compo- nent within the current calendar day. On first daily interaction, the system compares the current date against the user’s last recorded activity timestamp. If the gap is exactly one day, the streak increments; if the gap exceeds one day, the streak resets to one. This comparison is performed server-side using PHP’s Carbon date library to ensure timezone-independent accuracy.





Document Text Extraction Algorithm



When a user uploads a PDF or DOCX file, the backend invokes the appropriate parsing library based on the file type. The algorithm reads the binary file stream, decodes internal formatting layers, extracts raw text content, and presents it for folder assignment before persisting to the Notes table. (See Section 4.1.5 for the full design.)

WebSocket Message Broadcasting Algorithm



When a student sends a chat message, the Controller persists the message record and then broadcasts an event payload over a dedicated WebSocket channel. The receiver’s client-side listener dynamically appends the new message element without a page reload. (See Section 4.1.6 for the full design.)

Project Design

Strong system design ensures the product has room to grow, avoids architectural bottlenecks, and delivers a robust, responsive feedback loop to the end user.

Smart Document Parsing Design

The core conversion engine is designed to facilitate quick study note generation. When a doc- ument (such as a plain text string or basic document data) is provided into Mantra via the interface, a backend Laravel Controller translates the textual content into a standardized plain- text/HTML format, and subsequently injects this data into the creation workflow of a new Smart Note. The framework prompts the user with a customized modal dialog explicitly asking for the destination folder before officially migrating the parsed text to the PostgreSQL Notes table.



Figure 4.10: Code snippet detailing Note creation and XP/Streak gamification processing.



Real-time Notification Design

Web application polling is significantly detrimental to large scale chat systems. Therefore, Mantra’s message pipeline adopts the Publish/Subscribe (PubSub) design pattern using Web- Sockets. When User A shares a generated Smart Note with User B, the backend Controller commits the relational record to the DB, and simultaneously broadcasts an asynchronous event over the WebSocket channel. User B’s interface listens to this channel, dynamically spawning the message DOM element without ever having to refresh the active web page.





Project Pipeline

The Project Pipeline outlines the systematic development workflow followed during the cre- ation of the Mantra platform, from initial requirements gathering through final deployment and maintenance.

Requirements Analysis: Identified the core needs of the target user (students) — task management, note-taking, peer collaboration, and motivation through gamification. De- fined the functional and non-functional requirements that would guide all subsequent phases.

Testing: Executed comprehensive testing across Unit, Integration, System, and Perfor- mance layers to validate individual components, inter-module connectivity, end-to-end system behavior, and load resilience.

Version Control: Managed all source code changes through Git, maintaining a com- plete history of commits, feature branches, and iterative improvements throughout the development lifecycle.

Deployment: Packaged the Laravel application for production deployment on a Lin- ux/Windows server environment with PostgreSQL, ensuring all environment configura- tions, storage permissions, and WebSocket services are properly initialized.

Folder Structure

A clear and organized directory structure is essential for scaling and maintaining the Laravel application. Mantra’s codebase strictly follows the standard Laravel directory scaffolding, en- suring that controllers, models, views, and public assets are completely decoupled.









Key Directories

app/Http/Controllers: Contains all the business logic, including ChatController, NoteController, SmartNoteController, QuizController, EventController, TodoController, and the ProgressController managing XP and streaks.

resources/views: Houses all the Blade templates used to construct the frontend UI, in- cluding the Master layouts, Calendar partials, and Chat modals.

routes/web.php: Organizes all the web entry points connecting user requests to the ap- propriate backend Controllers.

public/images: The repository for static assets, including mantra.png and achieve- ment iconography like trophies and medals.

storage/app/public: Safely stores all user-uploaded documents (PDFs, DOCX files) be- fore they are processed by the Smart Notes extraction tools.

CONCLUSIONS



Mantra has successfully achieved its objective of reimagining the traditional digital learning en- vironment by tightly integrating essential productivity tools with engaging, gamified elements. Unlike disconnected applications where students must switch between a calendar app, a note repository, and an instant messenger, Mantra consolidates these workflows into a cohesive, centralized dashboard built on a robust Laravel foundation.

The incorporation of the Experience Points (XP) reward system, tracking daily study streaks, and visual achievements directly combat the lack of motivation frequently observed in modern remote learning. Concurrently, the Smart Notes module reduces the friction of transcribing or securely storing lecture materials, while the WebSocket-driven Chat interface promotes seam- less peer collaboration. By prioritizing an intuitive user interface utilizing HTML5/CSS3 and smooth, secure backend processes via PHP and Laravel, Mantra delivers a modern, highly effective educational companion that inspires continuous academic growth.



FUTURE WORKS



While Mantra currently offers a robust suite of tools for student productivity and collaboration, several future enhancements are planned to expand the platform’s capabilities:

AI-Powered Note Summarization: Integrating Large Language Models (LLMs) to au- tomatically generate concise summaries or flashcards from the text extracted via the Smart Notes document parser.

Group Collaboration Channels: Expanding the 1-on-1 WebSocket chat functionality into dedicated group channels, allowing entire study groups or project teams to commu- nicate and share notes collectively.

Advanced Analytics Dashboard: Providing users with deeper insights into their pro- ductivity trends over time, such as charting optimal study hours or highlighting subjects that require more focus.

Mobile Application Integration: Developing a companion iOS/Android mobile appli- cation to ensure students can manage calendars, track streaks, and chat seamlessly away from their desktop environments.

Customizable Themes: Allowing users to fully personalize their dashboard themes and layouts beyond the standard light/dark modes to further enhance the customized user experience.
